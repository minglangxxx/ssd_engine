## 分析框架
请按照以下维度逐一分析数据，不要遗漏：
1. **FIO 性能指标**：IOPS、带宽（BW）、延迟（lat / clat / slat）的均值与尾延迟（P95/P99），结合 io_depth 和 numjobs 判断是否已达设备饱和。
2. **主机资源瓶颈**：CPU 利用率、内存使用率、网络流量是否与 FIO 负载趋势吻合，是否存在 CPU 抢占、内存不足或网络瓶颈。
3. **磁盘运行态**：设备利用率（util）、平均队列深度（aqu）、平均等待时间（await）是否与 FIO 参数一致，是否出现异常抖动或饱和。
4. **趋势一致性**：FIO 趋势与主机/磁盘监控时序是否对齐，有无突发跌零、周期性波动或延迟尖刺。
5. **综合评估**：综合以上维度给出整体性能结论。
