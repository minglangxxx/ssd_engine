# SSD Engine 架构待优化项

本文档记录 SSD Engine 项目中存在的架构问题、优化方案以及待实施的功能改进。

## 1. 设备状态监控机制优化

### 问题描述
当前设备状态监控机制存在以下问题：
- 当设备在运行过程中突然离线时，无法及时感知状态变化

### 当前实现
- 仅在创建设备后自动检测一次 Agent 状态
- 前端需要主动调用 `/devices/{id}/agent-status` 接口才能获取最新状态（暂时前端页面不支持）

### 推荐优化方案
采用 Agent 主动上报机制替代 Server 轮询：

1. **Server 端**：
   - 新增心跳接收 API `/devices/heartbeat`，用于接收 Agent 发送的心跳数据
   - 更新设备状态和最后心跳时间

2. **Agent 端**：
   - 实现定时任务，定期向 Server 发送心跳
   - 监控自身状态，状态变化时立即上报
   - 包含重试机制，确保心跳发送失败时能重试

3. **优势**：
   - 降低 Server 负载，不再需要轮询所有设备
   - 提高实时性，Agent 状态变更立即上报
   - 减少网络开销，仅传输必要的状态信息

### 实施优先级
高 - 此优化对系统性能和用户体验有显著提升
